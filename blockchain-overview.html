Bitzal Core Blockchain Architecture & Consensus
Version 1.0 | March 2026

📖 Blockchain Overview

Bitzal Core utilizes a hybrid blockchain architecture that combines the security of Proof-of-Work (PoW) with the advanced functionality of a masternode network. This two-tier approach enables features such as instant transactions, optional privacy, and decentralized governance while maintaining the decentralization and security guarantees of traditional blockchain systems.

The Bitzal blockchain is designed for scalability, with 5MB blocks and a dynamic block reward system that ensures long-term sustainability. The network achieves finality through a combination of PoW confirmations and masternode quorum consensus.

Key Blockchain Specifications:
• Block Time: ~2.5 minutes
• Block Size: 5 MB (activated April 2026)
• Consensus: Proof-of-Work (X11 + MinotaurX) with Masternode Layer
• Total Supply: 84,000,000 ZAL
• Transaction Finality: InstantSend (2-3 seconds) + 6 block confirmations
• Privacy: Optional PrivateSend coin mixing

🏗️ Blockchain Architecture

1. Two-Tier Network Structure

Bitzal's architecture consists of two distinct layers that work together to provide security, functionality, and scalability:

⛏️ Tier 1: Proof-of-Work Miners

Responsible for securing the blockchain through computational work. Miners solve complex cryptographic puzzles to create new blocks, validate transactions, and earn block rewards. Bitzal supports two mining algorithms: X11 (ASIC) and MinotaurX (CPU/GPU).

🖥️ Tier 2: Masternode Network

A network of full nodes that hold 8,000 ZAL collateral. Masternodes provide advanced services including InstantSend, PrivateSend, governance voting, and network consensus. They receive 45% (X11) or 20% (MinotaurX) of block rewards.

⚙️ Consensus Mechanism

1. Proof-of-Work (PoW)

Bitzal uses a dual-algorithm Proof-of-Work system. Miners compete to find a hash below the current difficulty target. The winning miner receives block rewards and transaction fees.

MinotaurXCPU/GPU miners70% of block rewardDarkGravityWave (per block)

2. Masternode Consensus (DIP3)

Bitzal implements Deterministic Masternode Lists (DIP3), which provide:

  • Deterministic Masternode Registration — Masternodes are registered on-chain with their collateral transaction
  • Automatic Quorum Formation — Masternodes are automatically selected for quorums using a deterministic algorithm
  • LLMQ (Long-Living Masternode Quorums) — Quorums that persist for multiple blocks, enabling efficient consensus
  • InstantSend & ChainLocks — Quorum-based transaction locking for instant finality

3. DarkGravityWave (DGW) Difficulty Adjustment

Both mining algorithms use DarkGravityWave for difficulty adjustment. DGW recalculates difficulty every block using a moving average of the last 24 blocks, ensuring:

  • Fast response to hash rate fluctuations
  • Protection against time warp attacks
  • Smooth difficulty transitions
  • Consistent block times (~2.5 minutes)

📝 Transactions

1. Transaction Structure

Bitzal transactions follow the UTXO (Unspent Transaction Output) model, similar to Bitcoin but with additional features:

Transaction {
    Version: 2
    Inputs: [{
        Previous Output Hash
        Previous Output Index
        Signature Script
        Sequence Number
    }]
    Outputs: [{
        Amount (ZAL)
        Script (recipient address)
    }]
    LockTime: 0
    Extra Features: InstantSend | PrivateSend
}

2. Transaction Types

⚡ InstantSend Transactions

Transactions locked by masternode quorum consensus, achieving finality in 2-3 seconds. Ideal for point-of-sale and time-sensitive payments. Requires no additional fees.

🔒 PrivateSend Transactions

Optional coin mixing feature that breaks the link between transaction inputs and outputs. Uses a multi-stage mixing process coordinated by masternodes. Provides enhanced privacy for users who require it.

📋 Governance Transactions

Special transaction types for submitting proposals, voting on budgets, and distributing treasury funds. All governance actions are recorded on-chain.

👥 Masternode Registration

Transactions that lock 8,000 ZAL as collateral and register a masternode on the network. The collateral remains locked while the masternode is active.

3. Transaction Confirmation Process

  1. Broadcast: Transaction is broadcast to the network
  2. Mempool: Transaction enters the memory pool
  3. InstantSend (optional): If enabled, masternode quorum locks inputs (2-3 seconds)
  4. Block Inclusion: Miner includes transaction in a block (~2.5 minutes)
  5. Confirmations: Each subsequent block adds confirmation depth
  6. Finality: 6 confirmations recommended (~15 minutes)
Transaction Fees:
• Standard transactions: 0.0001 ZAL per KB
• InstantSend: No additional fee
• PrivateSend: 0.001 ZAL mixing fee (distributed to masternodes)

🛡️ Blockchain Security

1. Security Layers

  • Proof-of-Work: Computational security preventing double-spend attacks
  • Masternode Quorums: Instant transaction locking and ChainLocks preventing reorg attacks
  • Spork Technology: Multi-signature (3-of-5) feature activation preventing malicious upgrades
  • Deterministic Masternode Lists: Sybil resistance through collateral requirement
  • DarkGravityWave: Protection against difficulty manipulation
