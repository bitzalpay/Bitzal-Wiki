📖 Introduction

Bitzal utilizes a unique dual-algorithm Proof-of-Work system, combining the battle-tested X11 for ASIC miners with the innovative MinotaurX for CPU/GPU mining. This approach creates a balance between network security, mining accessibility for ordinary users, and protection against centralization.

Unlike most cryptocurrencies that rely on a single algorithm, Bitzal fosters an ecosystem where every participant can contribute to network security in the most efficient way for their circumstances.

⚙️ Section 1: X11 — The Proven ASIC Algorithm

1.1 History and Origins

X11 was first introduced in 2014 by the developers of Dash (then Darkcoin) as an innovative algorithm initially positioned as ASIC-resistant. The algorithm consisted of a chain of 11 different hashing functions, which was intended to complicate the creation of specialized hardware.

However, by 2016-2017, ASIC manufacturers (Baikal, iBeLink, Pinidea) successfully developed specialized integrated circuits for X11, transitioning the algorithm into the ASIC-dominated category. Despite this, X11 remains one of the most reliable and proven algorithms in the industry, used in dozens of cryptocurrencies.

1.3 Working Principle

Each data block passes sequentially through all 11 algorithms:

Input Data → Blake → BMW → Groestl → JH → Keccak → Skein → Luffa → Cubehash → Shavite → Simd → Echo → Output Hash

🔬 Section 2: MinotaurX — The Innovative CPU/GPU Algorithm

2.1 Origins and Philosophy

MinotaurX was developed in 2025 as a response to the problem of mining centralization. While X11 became accessible to ASICs, the cryptocurrency community desperately needed an algorithm that would allow ordinary users to participate in mining again using their home computers.

The name "MinotaurX" symbolizes: Minotaur — a mythical creature combining different forms (just as the algorithm combines CPU and GPU mining) and X — a reference to X11, emphasizing technological continuity.

2.2 MinotaurX Technical Specification

MinotaurX extends the X11 concept by adding a 12th algorithm and optimizing the entire chain for efficient CPU and GPU operation:

2.3 Key Features

⚡ CPU Optimization

AVX2/AVX-512 instructions, multithreaded processing, cache locality, vectorization for modern processors.

🎮 GPU Optimization

OpenCL/CUDA support, Wave32/Wave64 optimization, register pressure reduction, maximum compute unit utilization.

🛡️ ASIC Resistance

12-algorithm chain, heterogeneous operations, memory-intensive steps, branching, floating point operations.

🌊 Section 5: DarkGravityWave — Difficulty Adjustment Algorithm

Both mining algorithms use a unified difficulty adjustment algorithm — DarkGravityWave (DGW) — which provides smooth difficulty adjustment with each new block, protection against sudden hashrate spikes, and fair block distribution over time.

Difficulty_n = Difficulty_(n-24) * (2^24) / (2^24 - 1) + Target_Time / Actual_Time
DGW Advantages: Algorithm independence, fast response (adjustment on each block), manipulation protection (averaging eliminates outliers), proven stability in Dash and other coins.

7.2 Proof-of-Work Verification

// Example from src/pow.cpp
bool CheckProofOfWork(const CBlockHeader& block, const Consensus::Params& params) {
    POW_TYPE powType = block.GetPoWType();
    arith_uint256 bnTarget;
    bnTarget.SetCompact(block.nBits);
    uint256 hash = block.GetPoWHash();
    
    if (UintToArith256(hash) > bnTarget)
        return false;
        
    arith_uint256 bnLimit;
    if (powType == POW_TYPE_X11) {
        bnLimit = UintToArith256(params.powLimit);
    } else {
        bnLimit = UintToArith256(params.powTypeLimits[POW_TYPE_MINOTAURX]);
    }
    return bnTarget <= bnLimit;
}
